Method

AI Engineering Radar is a narrow engineering intelligence system, not a general AI newsletter. It is built for builders and researchers working on AI engineering, infra/inference, robotics, VLM/VLA, and adjacent deep tech — not a firehose of headlines.

Architecture

The site is fully static. A scheduled GitHub Actions workflow runs once every 24 hours, fetches structured sources (RSS/Atom, arXiv category feeds, GitHub release feeds), normalizes and deduplicates the items, scores them, builds the daily digest, and commits the generated JSON/HTML back into this repository. GitHub Pages then serves the updated site — there is no server, database, or login.

Pipeline stages

  • fetch_feeds.py — pulls entries from every source in config/sources.yaml
  • normalize.py — converts raw entries into a common item schema
  • dedupe.py — collapses exact-URL duplicates and near-duplicate titles into one item, keeping the highest-trust source as the representative
  • score.py — computes relevance, novelty, source trust, and engineering value
  • build_digest.py — selects the top items per section
  • export_pages.py — writes docs/data/*.json and docs/digest/*.html

Scoring formula

Each item gets a score in [0, 1]:

score = 0.40 × relevance + 0.25 × novelty + 0.20 × source_trust + 0.15 × engineering_value
  • relevance — how strongly the title/summary matches the taxonomy categories (model releases, infra/inference, robotics & VLA, multimodal, papers, repos, industry)
  • novelty — decays with age and drops to zero if the item already appeared in a recent digest
  • source_trust — a per-source weight reflecting how official/authoritative the source is
  • engineering_value — whether the item helps a builder (code, benchmarks, APIs, reproducible results) rather than just being a headline

Items below the minimum score threshold (see config/scoring.yaml) are dropped entirely. The remaining items are grouped into sections (Top Stories, Model & Tooling, Robotics & VLA, VLM & Multimodal, Papers & Benchmarks, Repo Watch, Industry Moves), each capped at a small number of items so the digest stays sharply filtered.

Deduplication

Items are merged if their canonical URL matches exactly, or if their normalized titles are similar enough (via a string-similarity threshold). When sources overlap, the highest-trust source is kept as the representative and the others are listed as "also reported by".

Update cadence

The pipeline runs on a daily schedule (plus manual trigger for testing). Each run produces a new docs/data/archive/<date>.json entry and updates docs/data/latest.json.

What this is not

  • Not a general AI newsletter — low-signal/hype content is filtered out
  • Not a chatbot — there is no runtime AI, only a scheduled content pipeline
  • Not a broad web scraper — sources are structured feeds (RSS/Atom, arXiv, GitHub releases)
